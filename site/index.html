Install and go

Six lines. No service to sign up for.

Give nominee a way to get a token. It handles freshness, approval, and audit on top.

agent.ts 
import { Nominee } from 'nominee'

// the whole setup — a function that returns a token
const nominee = new Nominee({
  strategy: ({ connection }) => process.env[`${connection.toUpperCase()}_TOKEN`]!,
})

// at tool-call time — always fresh, cached until just before expiry
const token = await nominee.token({ user: 'alice', connection: 'github' })
import { nomineeTool } from 'nominee-ai'
import { z } from 'zod'

const closeIssue = nomineeTool({
  nominee, user: 'alice',
  connection: 'github',   // fresh token injected into execute()
  approval: true,         // pause for human approval first
  description: 'Close a GitHub issue',
  inputSchema: z.object({ repo: z.string(), issue: z.number() }),
  async execute({ repo, issue }, { token }) {
    // `token` is fresh, auto-refreshed by nominee
  },
})
import { Nominee } from 'nominee'
import { Auth0 } from 'nominee-auth0'

// same API — swap the strategy for managed Token Vault + CIBA
const nominee = new Nominee({
  strategy: Auth0({
    domain: process.env.AUTH0_DOMAIN!,
    clientId: process.env.AUTH0_CLIENT_ID!,

    ciba: { loginHint: (user) => store.getSub(user) }, // phone approval
  }),
})

How it works

A letter of authority: issued, cached, revocable.

The user is the principal. Your agent is the nominee, authorized to act on their behalf within a scope. nominee fetches a fresh token at the moment of each call, gates the risky ones, and logs the chain.

Why nominee

Why not just use what you already have?

Your orchestrator won't touch third-party auth; one IdP solves it but owns your stack. The honest gap, side by side.

Orchestrators (AI SDK, Cloudflare Agents, LangChain) run the agent but by design hand you the third-party auth. Auth0 solves the mechanics beautifully, but you marry it. nominee is the neutral seam between them: start with a function, swap to Auth0 in one line.

When you don't need nominee: a single-provider, short-lived, non-durable app: use your IdP's SDK or ~20 lines of refresh. nominee earns its place when agents are long-running, durable, multi-provider, or you refuse lock-in.
